Serverless Web App on AWS

A minimal static site that showcases an example AWS architecture using Amazon S3, CloudFront, API Gateway, and Lambda.

View Architecture Deploy in Minutes

Overview

This demo represents a typical serverless pattern: a static frontend hosted on S3, cached globally via CloudFront, interacting with a secure API behind API Gateway and AWS Lambda, with DynamoDB for persistence and Amazon Cognito for auth (optional).

  • S3 Static Website Hosting
  • CloudFront CDN
  • API Gateway + Lambda
  • DynamoDB
  • Cognito (optional)
  • IAM + CloudWatch

Architecture

Why this pattern?

It provides scalability, global performance, and a pay-per-use backend. Static assets are served at low cost via S3+CloudFront, while dynamic requests scale via Lambda/APIGW.

Deploy to S3 + CloudFront

      
    
        
  1. Create an S3 bucket (e.g., my-aws-demo-site) and enable static website hosting.
    2. 
        
  2. Upload index.html, styles.css, and script.js to the bucket.
    3. 
        
  3. Set bucket policy to allow public read for objects (or use CloudFront OAI/Origin Access Control for private buckets).
    4. 
        
  4. Create a CloudFront distribution with the S3 bucket as the origin. Enable TLS and caching for /*.css, /*.js, and images.
    5. 
        
  5. (Optional) Point your domain to CloudFront via Route 53 and request an ACM certificate.
    6. 
      

      
Replace the API_BASE in script.js with your API Gateway endpoint when you have one.
